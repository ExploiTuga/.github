# Contribuir para o exploiTuga

Obrigado pelo interesse em contribuir para os projetos do **exploiTuga**.

A nossa comunidade acredita que a partilha de conhecimento, a colaboração e o desenvolvimento open source são fundamentais para o crescimento da comunidade de cybersecurity.

Aceitamos contribuições de diferentes tipos, incluindo código, documentação, investigação, correções, testes e ideias.

---

## 🧑‍💻 Como contribuir

Antes de contribuir para um projeto:

1. Consulta o `README.md` do projeto.
2. Consulta o `CONTRIBUTING.md`, caso exista uma política específica.
3. Consulta o `SECURITY.md` antes de reportar vulnerabilidades.
4. Verifica as Issues existentes para evitar trabalho duplicado.
5. Abre uma Issue quando for necessário discutir uma alteração significativa.

---

## 🔀 Pull Requests

As contribuições de código devem ser submetidas através de Pull Requests.

Um Pull Request deve:

* Ter uma descrição clara da alteração.
* Explicar o problema que pretende resolver.
* Incluir testes quando aplicável.
* Não introduzir secrets, credenciais ou dados sensíveis.
* Seguir as convenções de código do projeto.
* Passar os testes automatizados configurados no projeto.

Todas as alterações podem ser sujeitas a revisão antes de serem integradas.

---

## 🐛 Issues

Utiliza as Issues para:

* Reportar bugs.
* Sugerir funcionalidades.
* Propor melhorias.
* Discutir problemas técnicos.
* Propor novas ideias para projetos.

Antes de abrir uma Issue, verifica se já existe uma Issue relacionada.

---

## 🔐 Vulnerabilidades

Não utilizes Issues públicas para reportar vulnerabilidades de segurança.

Para reportar uma vulnerabilidade, consulta o ficheiro `SECURITY.md` do respetivo projeto e segue o processo de divulgação responsável definido nesse projeto.

---

## 📚 Documentação

Contribuições para documentação são igualmente importantes.

Podes contribuir com:

* Tutoriais.
* Guias técnicos.
* Write-ups.
* Research.
* Correções de erros.
* Melhorias de documentação.
* Exemplos práticos.

---

## 🤝 Código de Conduta

Todas as contribuições devem respeitar o nosso [Código de Conduta](CODE_OF_CONDUCT.md).

O objetivo é manter uma comunidade aberta, inclusiva, profissional e focada na partilha de conhecimento.

---

## ⚖️ Responsabilidade

Ao contribuir para um projeto do **exploiTuga**, assumes a responsabilidade de garantir que o teu código, conteúdo ou investigação não viola direitos de terceiros ou legislação aplicável.

As ferramentas e técnicas publicadas devem ser utilizadas apenas em sistemas próprios ou onde exista autorização explícita para realizar testes de segurança.

---

Obrigado por contribuíres para o **exploiTuga**.

**Explorar. Aprender. Investigar. Construir.**
