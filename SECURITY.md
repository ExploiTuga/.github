# Política de Segurança

A segurança dos projetos do **exploiTuga** é uma prioridade.

Se encontrares uma vulnerabilidade de segurança num dos nossos projetos, agradecemos que a reportes de forma responsável, permitindo-nos investigar e corrigir o problema antes de qualquer divulgação pública.

---

## 🔐 Reportar uma vulnerabilidade

Não utilizes GitHub Issues públicas para reportar vulnerabilidades.

Sempre que possível, utiliza a funcionalidade **Private Vulnerability Reporting** do GitHub disponível no respetivo repositório.

Caso esta funcionalidade não esteja disponível, contacta diretamente os maintainers do projeto através dos canais oficiais indicados no respetivo repositório.

---

## 📋 Informação recomendada

Ao reportar uma vulnerabilidade, inclui:

* Descrição detalhada do problema.
* Projeto e versão afetada.
* Passos necessários para reproduzir o problema.
* Impacto potencial.
* Prova de conceito, quando apropriado.
* Sugestões de mitigação, se disponíveis.

Evita incluir dados pessoais ou informação sensível desnecessária.

---

## ⏱️ Processo de resposta

Após recebermos um relatório:

1. O relatório será analisado.
2. A vulnerabilidade será validada.
3. O impacto será avaliado.
4. Será desenvolvida uma correção quando aplicável.
5. A correção será testada.
6. Será preparada uma nova versão do projeto quando necessário.
7. A vulnerabilidade poderá ser divulgada de forma responsável após a correção.

---

## 🛡️ Divulgação responsável

Pedimos aos investigadores que permitam um período razoável para investigação e correção antes de divulgarem publicamente uma vulnerabilidade.

Reconhecemos e valorizamos o trabalho de investigadores de segurança que reportem vulnerabilidades de forma responsável.

---

## ⚠️ Scope

Esta política aplica-se aos projetos mantidos pela organização **exploiTuga**, salvo indicação em contrário no repositório específico.

Cada projeto pode definir requisitos adicionais na sua própria política de segurança.

---

## 📜 Legal

Testes de segurança devem ser realizados exclusivamente em sistemas para os quais exista autorização adequada.

O **exploiTuga** não autoriza nem incentiva atividades ilegais ou testes não autorizados contra sistemas de terceiros.
